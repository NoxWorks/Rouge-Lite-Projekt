# Roadmap – Roguelite-Projekt

## Phase 1: Grundfunktionen und Engine-Struktur

### Gameplay Phase1

- Spielerbewegung und Kollision implementieren
- Erste Gegnerlogik (Nahkampf, Fernkampf, Flucht)
- Dungeon-Generator mit zufälligen Räumen, Raumgrößen und Geheimräumen
- Raumtypen definieren (Kampf, Shop, Boss, Event, Secret, Start)

### Technik Phase 1

- Scene-Management vorbereiten
- GDScript-Dateien organisieren
- Input-Handling für Tastatur und/oder Controller
- Debug-Overlay oder Logging-Funktion

### Assets Phase 1

- Platzhalter-Sprites für Spieler und Gegner
- Tileset für Dungeon erstellen

## Phase 2: Spielsysteme und Erweiterungen

### Gameplay Phase 2

- Hub-Bereich erstellen
- Item-Crafting-System entwickeln
- Item-Upgrades und Charakter-Upgrades
- „Level“-Modifikatoren einbauen
- Item-Rarity und Item-Pools definieren

### Technik Phase 2

- Save/Load-System vorbereiten
- Gegnerverhalten verfeinern

### Assets Phase 2

- UI-Elemente (Lebensanzeige, Inventar)
- Menüs und HUD gestalten
- Hub visuell designen

## Phase 3: Dokumentation und Präsentation

- GitHub und Organisation
- README erweitern mit Screenshots oder Gifs
- Projektstruktur auf GitHub dokumentieren
- Devlog starten (devlog.md oder ein Youtube Kanal)
- Versionsnummern und Meilensteine festlegen
- Screenshots regelmäßig dokumentieren
